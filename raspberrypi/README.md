### raspberry pi 本体のセットアップ

○ 手順

1. raspberry pi 5 付属の micro ssd を mac pc 本体に接続（ここで micro ssd → type c 変換アダプタが必要）

2. 公式の raspberry pi 5 の imager ページ（https://www.raspberrypi.com/software/）から PC os に対応したイメージをダウンロード

3. Downloads またはデフォルトの保存フォルダからユーザルートの Applications にイメージャの.dmg ファイルを移行

4. イメージャの.dmg ファイルをダブルクリックして専用の GUI を起動すると、左側にラズベリーみたいなのがあり、右側にフォルダがある。左側のラズベリーをダブルクリックして、なんか許可しますか？と言われたら「はい」で進む。

5. その後画面が変わる？新しい gui インスタンスが起動するので、それの最初の画面で、Raspberry pi のモデル指定、OS のモデル指定、ストレージ指定（これはファイルシステムにマウントされている untitled の micro ssd の方のストレージを選択）

6. 続けてそのまま選択を進めると、Raspberry pi 基盤本体にデータの移行、初期化が開始される。終わるまで待機。

（実際の作業内容に基づいて質問をした AI からの回答）

Raspberry Pi Imager で SSD に Raspberry Pi OS (Debian 系 64bit)を書き込む際、「既存のデータを消すか（フォーマットして良いか）」は必ず Yes で進める必要があります。これは書き込みが「ディスク全体の初期化および OS イメージ展開」だからです。既存データは消去されます。

また、Imager のカスタム設定機能は限定的です。例えば「Wi-Fi 設定」「SSH 有効化」「ユーザー名・パスワード決定」など基本的な項目のみしか画面 UI から設定できず、より詳細・複雑なカスタマイズ（例: インストール済みソフト追加やパーティション最大化設定など）は Imager インターフェースではできません。

高度なカスタム設定を行いたい場合は、下記の方法が一般的です。

書き込み後に Raspberry Pi 起動 →SSH 接続し、コマンドで設定変更やソフトインストールを行う。

Debian イメージを PC 上でマウントし（Linux 環境推奨）、chroot で環境に入り込んで直接カスタマイズする。

rpi-imager の「Use custom」機能で事前に設定できる最小限のカスタマイズは実行するが、それ以上は起動後の操作で対応。

要は、「SSD の既存データは消すことが前提」「Imager による OS 書き込みは基本的に OS イメージ全体を直接上書きする操作である」「より複雑なカスタムは OS 起動後に行う」のが通常の流れです。

このため、Imager で「カスタム設定がほとんどできなかった」ことは仕様として正常であり、前述手順通り書き込み後の起動・SSH 接続して環境構築を進めることを推奨します。
